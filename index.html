Competetive Snake

Explanation of the Task

If you don't know snake click here.

So the Idea is: the code you're submitting plays snake on the same board with another program. Your goal is to somehow kill the other snake.

The interesting part is that we had to modify this a bit to make it more fun:

  • Apples spawn differently. They spawn exactly every $20$ turns. At the beginning there already is an apple. They spawn random. They don't despawn until eaten. (This makes the game end at some point.)
  • If your head and the other snakes head collide directly (Meaning you're going on the same field) then the longer snake wins. If they have the same length then it's a draw.
  • If there's no more space to spawn an apple the longer snake wins. If they have the same length then it's a draw.
  • You're starting only with the snakes head. (No tail. Yes that means you can just turn on the spot as long as you keep this length of $1$.)

You're always starting in the top left corner. (This works by the same magic trick responsible for you never being the red party in Clash Royale.)

Point system

Your programs compete in a similar way like in football leagues but since we're mathematicians: You don't get $3$ point for winning you get $\pi$ points. And you don't get $1$ point for a draw you get ${e^i}^i$ points. And another important difference is that you're not competing with everyone exactly once you do it a hundred times each on different sized boards.

Explanation of the input and output format.

At the beginning of a new game your program is executed as a new process. Meaning you can't remember previous games. (Yes: you can't train a machine learning algorithm in the tournament) But indeed you can remember the previous rounds in the current game you're playing.
Your program gets all the input via stdin. So for example in python you can get in using input() and in C++ you read it using std::cin. You can output via stdout. So for example in python you can use print() and in C++ you can output using std::cout.

The first input your code gets is the size of the board: two numbers $w$ and $h$ indicating the width and the height of the board. Formatted like:
w h
They are seperated by a space and afterward comes a newline ("\n"). For example if the board is $6$ blocks wide and $3$ blocks tall then this line is:
6 3

Then the next in formation you get is the current board. It's basically a $w\times h$ matrix of integers each entry representing a square on the board. If this integer is $0$ then the field is empty.

If it is an odd number it's part of your snake. These numbers are increasing from the head on. This means: The $1$ represents your head. The $3$ represents the part of your snake adjacent to your head. The $5$ represents the following piece and so on and so on.

Equivalently the even numbers represent the parts of the snake of your opponent. These numbers are also increasing from the head on. This means: The $2$ represents your opponent's head. The $4$ represents the part of your opponent's snake adjacent to its head. The $6$ represents the following piece and so on and so on.

$-1$ is an apple.

So the format looks like this:
v_11 v12 ... v1(w-1) v1w
v_21 v22 ... v2(w-1) v2w
...
v_()1 v12 ... v1(w-1) v1w
v_11 v12 ... v1(w-1) v1w

Then your program has to return the direction it wants to move to:

  • d for down
  • u for up
  • l for left
  • r for right

IMPORTANT: you always have to send a direction also if you do not steer. And you always have to print a newline ("\n") afterward. (Note that using print in python does this automatically).

After you sent this, again your program gets the current state of the board. And you again have to send the direction you want to move to. Then you're sent the board and so on and so on.

When the game ends your code gets nothing back or something like this your program is just terminated.

Example Code

Python

The following python code collaborates with all other prisoners: 
# reading number of prisoners (n) and the own assigned number (k)
n, k = map(int, input().split())

# repeating for every round
while (True):
    # printing all numbers 1 to n-1 without the own number k
    # to indicate one is collaborating with all other prisoners
    print(" ".join(map(str,set(range(n)).difference({k}))))
    # reading the n lists who collaborated with whom
    for i in range(n):
        # reading the line of the list of lists
        line_numbers = list(map(int, input().split()))
        # splitting the read numbers in l the count of prisoners the i-th
        # prisoner collaborated with and collabs the numbers of the prisoners
        # the i-th prisoner collaborated with
        l, collabs = line_numbers[0], line_numbers[1:]
        # ignoring the read information
The following python code collaborates with no one: 
# reading number of prisoners (n) and the own assigned number (k)
n, k = map(int, input().split())

C++

The following C++ code collaborates with all other prisoners: 
#include <iostream>

int main(){
    // reading number of prisoners (n) and the own assigned number (k)
    int n, k;
    std::cin >> n >> k;

    // repeating for every round
    while(true){
        // iterating over all n prisoners
        for (int i = 0; i < n;i++){
            // if not oneself printing i to indicate that one is collaborating
            // with the i-th prisoner
            if (n!=k)
                std::cout << i << " ";
        }
        // printing end of line ("\n") to indicate the end of the list
        std::cout << std::endl;
        // reading for each prisoner with whom they collaborated
        for (int i = 0; i < n;i++){
            // reading the count l of prisoners the i-th prisoner
            // collaborated with
            int l;
            std::cin >> l;
            // reading all l prisoners the i-th prisoner collaborated with
            for (int j = 0; j < l; j++){
                // reading the number c of one the l prisoners the i-th
                // prisoner collaborated with
                int c;
                std::cin >> c;
                // ignoring the read information
            }
        }
    }
}
